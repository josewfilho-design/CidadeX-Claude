📱 Guia de Publicação — CidadeX

Google Play Store • Versão 1.4.4 • Fevereiro 2026

1. Pré-requisitos

  • Conta de Desenvolvedor Google Play ($25 taxa única) — play.google.com/console
  • Android Studio instalado (versão mais recente)
  • Node.js 18+ e npm instalados
  • Git instalado e projeto clonado do GitHub
  • JDK 17 instalado (necessário para builds Android)

2. Configuração do Projeto

2.1 Dados do Capacitor

CampoValor
App IDcom.cidadex.br
App NameCidadeX
Web Dirdist
⚠️ Importante — Produção Remova o bloco server do capacitor.config.ts antes do build de produção. O app deve servir arquivos locais, não de um servidor remoto.

2.2 Comandos Iniciais

# 1. Clonar o repositório
git clone https://github.com/SEU_USUARIO/cidadex-br.git
cd cidadex-br

# 2. Instalar dependências
npm install

# 3. Adicionar plataforma Android (apenas primeira vez)
npx cap add android

# 4. Gerar ícones nativos (opcional — requer @capacitor/assets)
npm install -D @capacitor/assets
npx capacitor-assets generate --iconBackgroundColor "#1a1f36" --splashBackgroundColor "#1a1f36"

# 5. Build de produção
npm run build

# 6. Sincronizar com Android
npx cap sync android

# 7. Abrir no Android Studio
npx cap open android

3. Gerar App Bundle Assinado (.aab)

3.1 Criar Keystore (apenas primeira vez)

  1. No Android Studio: Build → Generate Signed Bundle / APK
  2. Selecione Android App Bundle
  3. Clique em Create new... para criar uma nova Keystore
  4. Preencha os campos:
    CampoExemplo
    Key store path~/cidadex-keystore.jks
    PasswordSenha forte (anote e guarde!)
    Key aliascidadex-key
    Key passwordSenha forte (anote e guarde!)
    Validity (years)25
    First and Last NameSeu nome completo
    OrganizationCidadeX
    Country CodeBR
  5. Clique OK e depois Next
  6. Selecione release e clique em Create
🔐 CRÍTICO — Guarde a Keystore! A Keystore é a identidade do seu app. Se perdê-la, nunca mais poderá atualizar o app na Play Store. Faça backup em local seguro (pen drive, cofre, nuvem criptografada).

3.2 Localização do Bundle Gerado

android/app/release/app-release.aab

5. Declarações Obrigatórias

Antes de publicar, a Google exige que você preencha várias declarações no painel. Complete cada seção:

  • Política de Privacidade — URL obrigatória (use a página /privacy do app publicado)
  • Acesso ao app — Se o app requer login, forneça credenciais de teste
  • Anúncios — Declare se o app contém anúncios (CidadeX: Não)
  • Classificação do conteúdo — Preencha o questionário IARC
  • Público-alvo — Selecione a faixa etária (13+ recomendado)
  • App de notícias — Declare se é um app de notícias
  • Segurança dos dados — Declare os dados coletados/compartilhados
  • Apps governamentais — Declare que NÃO é um app governamental

6. Publicação e Revisão

  1. Vá em Produção → Criar nova versão
  2. Faça upload do arquivo .aab
  3. Adicione notas da versão: 
    Versão 1.4.4
• App completo para cidades do Ceará
• Notícias, eventos e clima em tempo real
• Navegação GPS com mapas interativos
• Agenda pessoal e controle financeiro
• Chat e rede social local
• Assistente IA integrado
  4. Clique em Revisar versão e depois Iniciar lançamento para Produção
⏱ Tempo de Revisão A primeira revisão pode levar de 3 a 7 dias úteis. Atualizações subsequentes costumam levar 1-2 dias.

  • Bloco server removido do capacitor.config.ts
  • Build de produção gerado (npm run build)
  • Sincronização feita (npx cap sync android)
  • Keystore criada e guardada em local seguro
  • App Bundle (.aab) gerado e assinado

  • Público-alvo definido
  • Versão enviada para revisão

9. Solução de Problemas Comuns

ProblemaSolução
Tela branca no appVerifique se o bloco server foi removido e se o npm run build rodou sem erros
Ícone genérico do AndroidRode npx capacitor-assets generate e depois npx cap sync
Erro de assinaturaVerifique se está usando a mesma Keystore e as senhas corretas
Rejeição por políticaRevise as declarações na seção 5 e corrija os pontos indicados pelo Google
versionCode duplicadoIncremente o versionCode no build.gradle
App crash na aberturaTeste em emulador antes. Verifique o Logcat no Android Studio
